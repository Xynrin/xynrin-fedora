## 仓库全貌

```
fedora-setup/
├── install.sh             # 主入口：mirrors → repos → dnf → flatpak → fish → fisher → vscode → node → systemd → scripts
├── mirrors/               # dnf 镜像切换（清华/中科大/阿里/官方）
├── repos/                 # 第三方 dnf repo 文件 + COPR 清单
├── packages/              # dnf + flatpak 包清单
├── fish/                  # config.fish / plugins / 函数 / universal 变量快照
├── vscode/                # settings.json + 扩展清单
├── kde/                   # Plasma 配置双向同步（pull.sh / push.sh）
├── node/                  # npm 全局包清单
├── systemd/               # 要启用的系统/用户服务
├── scripts/               # 命令行工具（自动 symlink 到 ~/.local/bin）
└── docs/                  # 这份文档
```

**核心理念**：
- `install.sh` 里的每一步都是**幂等**的（状态先查后动），重复跑没副作用
- 配置文件走 **symlink** 到家目录，改一处两边同步
- Plasma 的 rc 文件走 **双向快照**（pull/push），因为它被系统频繁改写，用 symlink 会让 git 工作区永远脏
- universal 变量（tide 颜色、fish_color_\*）用**生成式快照**，调完跑 `dump-fish-vars.fish` 重新导出

## 日常场景

### 场景 1：新机从零开始

假设刚装好 Fedora KDE spin，没登过 Plasma（避免默认配置覆盖）。

```sh
# 1. 准备 git
sudo dnf install -y git

# 2. 克隆仓库
git clone https://github.com/Xynrin/fedora-setup.git ~/fedora-setup
cd ~/fedora-setup

# 3. 一键跑
./install.sh
```

跑完后：
```sh
# 4. 部署 KDE 配置（登录 Plasma 前做效果最好）
./kde/push.sh

# 5. 登录 Plasma，或者：
kquitapp6 plasmashell && kstart plasmashell
```

akmod-nvidia 装完建议重启让模块生效。

---

### 场景 2：更新系统

fish 里敲：
```sh
up                    # dnf upgrade + autoremove + flatpak update
```

fisher 插件更新（注意 GitHub 匿名 API 每小时 60 次限制，有被限流风险）：
```sh
fisher update
```

---

### 场景 3：我在 Plasma 里改了外观，想保存

```sh
~/fedora-setup/kde/pull.sh
cd ~/fedora-setup
git diff kde/
# diff 看着对就：
git add kde/
git commit -m "kde: 更新面板布局和配色"
git push
```

---

### 场景 4：我调了 tide 颜色 / fish 语法高亮

```sh
fish ~/fedora-setup/scripts/dump-fish-vars.fish
git -C ~/fedora-setup diff fish/universal_vars.fish
git -C ~/fedora-setup commit -am "fish: 更新 tide 配色"
```

---

### 场景 5：我装了新的 dnf 包，想加入管理

编辑 `packages/dnf.txt`，追加一行（带注释分类），然后：
```sh
./install.sh --only dnf --dry-run   # 看会不会漏装
```

---

### 场景 6：我装了新的 VSCode 扩展

```sh
code --list-extensions | grep <关键字>   # 找到扩展 id
```
把 id 追加到 `vscode/extensions.txt`。

---

### 场景 7：在另一台机器上同步最新配置

```sh
cd ~/fedora-setup
git pull
./install.sh             # symlink 会自动更新，包会补装
./kde/push.sh            # 部署 KDE 配置（需要重登生效）
```

## 扩展

### 写一个新的 fish 函数

```fish
# ~/fedora-setup/fish/functions/myfunc.fish
function myfunc --description '说明'
    # ...
end
```
然后：
```sh
./install.sh --only fish
```
新终端直接可用，tab 补全会识别 `--description` 里的说明。

### 写一个新的 shell 命令

```sh
# ~/fedora-setup/scripts/mytool.sh
#!/usr/bin/env bash
# ...
```
然后：
```sh
./install.sh --only scripts
```
会被 symlink 到 `~/.local/bin/mytool`（**脱掉 .sh 后缀**），直接输 `mytool` 即可调用。

### 加新的第三方 dnf 仓库

1. 在新机上先手动 `sudo dnf install .../release.rpm` 或 `sudo curl -o /etc/yum.repos.d/foo.repo ...`
2. 把 `/etc/yum.repos.d/foo.repo` 复制到仓库 `repos/foo.repo`
3. 加到 `packages/dnf.txt`（这个仓库提供的包）
4. `./install.sh --only repos --only dnf`

### 加新的 COPR

```sh
echo "owner/repo-name" >> ~/fedora-setup/repos/copr.txt
./install.sh --only repos
```

### 加 systemd 服务

写路径到 `systemd/system.txt`（系统级）或 `systemd/user.txt`（用户级）：
```
my.service
my.timer
```
然后 `./install.sh --only systemd`。

### 加 flatpak 应用

```sh
flatpak search xxx              # 找到 application id
echo "com.example.App" >> ~/fedora-setup/packages/flatpak.txt
./install.sh --only flatpak
```

### 加 KDE 配置文件

编辑 `kde/files.txt`（相对 `~/.config/`）或 `kde/konsole-files.txt`（相对 `~/.local/share/konsole/`），加一行，然后：
```sh
./kde/pull.sh
```

## 排错

### `install.sh` 某步 sudo 失败

在后台/非 TTY 环境下 sudo 无法读密码。解决：
```sh
# 在 fish 里直接跑，让 sudo 能弹密码
~/fedora-setup/install.sh
```
或者先 `sudo -v` 缓存凭证，再 5 分钟内跑。

### `fisher update` 卡住 / 返回 403

GitHub 匿名 API 限流（每小时 60 次）。等一小时，或配置 `GH_TOKEN` 环境变量后重试。

### VSCode 打开后没主题

检查 symlink 是否指向真实文件：
```sh
ls -la ~/.config/Code/User/settings.json
readlink -f ~/.config/Code/User/settings.json
head -3 ~/.config/Code/User/settings.json
```
如果 `head` 失败说明目标不存在；看 `~/fedora-setup/vscode/settings.json` 是否存在，有问题就 `./install.sh --only vscode` 重新 link。

### Plasma 加载异常，想回滚

```sh
ls ~/.config/.kde-backup-*   # 看最近的备份
# 手动 cp 回来或者：
cd ~/.config
cp -a .kde-backup-20260510-120000/* .
```

### tide prompt 显示方块或错位

检查终端字体：`Cascadia Code NF` 或其他 Nerd Font。装字体：
```sh
sudo dnf install -y cascadia-code-nf-fonts
```
然后重启终端。

### `./install.sh --only xxx` 识别不出

只接受：mirrors, repos, dnf, flatpak, fish, fisher, vscode, node, systemd, scripts。值写错会直接报 "unknown --only value"。

### akmod-nvidia 装了但没生效

```sh
sudo akmods                      # 强制重新构建内核模块
sudo dracut --force              # 重建 initramfs
sudo reboot
```

### 镜像切换后 `dnf install` 报 "无法找到包"

某些国内镜像对 rpmfusion 路径不同。临时切回官方：
```sh
sudo ~/fedora-setup/mirrors/switch-mirror.sh official
```
或者手动改 `mirrors/preferred.txt` 换一家（tuna / ustc / aliyun）。

## 安全注意

### 绝对不要提交的

`.gitignore` 已经覆盖常见情况，但手动操作时依然要警惕：

- `.env` / `*.key` / `*.pem` / `id_rsa*` / `id_ed25519*`
- `~/.ssh/` 下的任何东西（`ssh/` 已被 gitignore）
- `~/.gitconfig`（通常有 user.email，`git/config` 已被 gitignore）
- `~/.config/Code/User/globalStorage/` 下的 token

### 提交前自查

```sh
cd ~/fedora-setup
git diff --cached           # 看清楚每一行
git status --short          # 看清楚每个文件
```

怀疑有敏感内容，宁可放弃 commit 也不要推出去：
```sh
git reset HEAD <file>
```

## 设计决策

### 为什么 fish 函数用 symlink，KDE 用 pull/push？

- **fish 函数** 是**手写**的，用户改的次数远多于程序改——symlink 让编辑即生效、tab 补全自动走
- **Plasma rc** 被**程序**频繁改写（拖个图标、窗口位置变化都触发），symlink 会让 `git status` 永远有噪声；快照模式下用户主动决定"我要保存当前状态"

### 为什么 `universal_vars.fish` 用生成式快照，不直接存 `fish_variables`？

`fish_variables` 是二进制-ish 的内部格式，不适合 diff；生成式脚本（`set -U ...`）可读、可 merge、可手改。

### 为什么镜像站切换走独立脚本？

- 可以脱离 `install.sh` 单独使用：`switch-mirror.sh aliyun` 立刻切
- 幂等：每次从 `.mirror-backup/` 的原始 repo 重建，不会让 sed 副作用累积
- 一键回滚：`switch-mirror.sh official`

### 为什么 `install.sh` 不跑 `kde/push.sh`？

KDE 配置部署是破坏性的（需要注销重登），要由用户显式触发。`install.sh` 保持"装好就是装好"的语义，不做需要重登的事。

### 为什么 scripts 自动脱 `.sh` 后缀？

命令行审美：敲 `opt` 比 `opt.sh` 短；同时保留源文件扩展名让编辑器识别语法。
